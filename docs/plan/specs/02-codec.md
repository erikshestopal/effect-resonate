# 02 — Codec (`Codec.ts`)

## Objective

The value-encoding boundary: `ResonateCodec` + `ResonateEncryptor` services with a
default codec byte-compatible with the native TS SDK.

## Dependencies

## Key facts

- Native encoding: `undefined` → `{data:"", headers:{}}`; else JSON.stringify with a
  replacer special-casing `Number.POSITIVE_INFINITY`/`NEGATIVE_INFINITY` →
  `"__INF__"`/`"__NEG_INF__"`, `Error` → `{__type:"error", message, stack, name}`,
  `AggregateError` → `{__type:"aggregate_error", message, stack, name, errors}` —
  then base64 of the whole JSON string into `Value.data`. Decode reverses with a
  reviver reconstructing real `Error`/`AggregateError` instances.
- MUST: same codec path for `param` AND `value` including rejections.
- MUST: headers accompany data. Additionally write `resonate:schema` header naming
  the payload schema where known (additive; other SDKs ignore).
- Encryptor: byte-level transform after encode / before decode; `layerNoop` default.
- Schemas sit ABOVE the codec: schema-encode first, plain JSON through the codec.

## Deliverables

- `ResonateCodec` service + `layerJson` (native-compatible default).
- `ResonateEncryptor` service + `layerNoop`.
- Typed `EncodingError` failures tagged with direction (encode/decode) and context.

## Tests

- Byte-compatibility fixtures: encode values here, decode with expectations captured
  from the native codec (numbers, strings, objects, undefined, ±Infinity, Error,
  AggregateError, nested errors) — and the reverse direction.
- Rejection round-trip: an encoded Error decodes to an Error instance with
  message/name/stack preserved.
- Property: for JSON-representable values, decode(encode(v)) deep-equals v.
- Custom encryptor layer applies around the codec in both directions.

## Notes (implementation decisions)

- The native codec is expressed as ONE Schema codec (`ValueFromUnknown`:
  `unknown ⇄ Protocol.Value`): a recursive `Schema.Union` models the native
  JSON tree (`__type` markers ⇄ `Error`/`AggregateError` via
  `Schema.instanceOf`, Infinity sentinels, arrays/records recursing via
  `Schema.suspend`), piped through `Schema.fromJsonString` and
  `Schema.StringFromBase64`; `undefined ⇄ {headers:{}, data:""}` is a union
  member. No hand-rolled replacer/reviver; values inhabiting schema types are
  built with `.make()` constructors, never annotated or cast.
- Byte-compat fixtures in `test/Codec.test.ts` were generated by running the
  actual native codec (`repos/resonate-sdk-ts/src/codec.ts`) under bun; Error
  fixtures pin `stack` strings so encoding is deterministic.
- Known corner-case divergences from native (degenerate inputs, deliberate):
  a top-level bare function/symbol fails with a typed `EncodingError` instead
  of native's silent `{data:""}`; a malformed `aggregate_error` marker
  (missing/non-array `errors`) decodes as a plain object instead of throwing;
  non-`Date` class instances with prototype `toJSON` are destructured rather
  than serialized via `toJSON`. `Date` leaves pass through so `JSON.stringify`
  applies `toJSON`, as native.
- The encryptor is composed around the codec by `encodeValue`/`decodeValue`
  (the analog of the native `Codec` class); `decodeValue` guards empty data
  (via `SchemaParser.is`) BEFORE decrypting, matching native order.
- The `resonate:schema` header is provided as `withSchemaHeader`; writing it
  where payload schemas are known happens in specs 11/12/17 (the codec service
  signature takes no schema, per the binding DESIGN interface).
